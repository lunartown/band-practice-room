# 합주실닷컴 — 디자인 전면 리디자인 요청 프롬프트

> 이 문서는 클로드(디자인)에게 "현재 UI를 버리고 밑바닥부터 새로 그려달라"고 요청하기 위한 브리프입니다.
> **[고정]** 표시된 항목은 절대 바꾸지 마세요. 그 외 비주얼(레이아웃·타이포 스타일·컴포넌트 모양·여백·질감·무드)은 전부 새로 제안하세요.

## 0. 한 줄 요약

**"합주실(밴드 연습실)의 빈 예약 시간을 한 곳에서 빠르게 비교 검색하는 모바일 웹 서비스"** 의 UI를 처음부터 다시 디자인해 주세요.

## 1. 이 서비스가 뭔지 (전체 설명)

밴드를 하는 사람들은 합주실을 빌려서 연습합니다. 그런데 합주실 예약은 대부분 네이버 예약 등 외부 페이지에 흩어져 있어서, "이번 주 토요일 저녁에 강남에서 2시간 연속으로 빈 방"을 찾으려면 합주실마다 일일이 들어가 봐야 합니다.

**합주실닷컴**은 여러 합주실의 예약 가능 시간을 미리 수집해두고, 사용자가 **날짜 / 시간대 / 지역 / 필요한 연속 시간 / 인원** 조건으로 한 번에 비교해서, 빈 시간을 가진 합주실 목록을 보여줍니다. 예약 자체는 우리 사이트에서 하지 않고, 원래 예약 페이지(네이버 예약 등)로 링크아웃합니다.

- 서비스명: **합주실닷컴** (도메인 hapjusil.com), 로고는 🎸 + "합주실닷컴" 워드마크
- 태그라인: **"합주실 빈 시간, 한 곳에서"**
- 핵심 가치: **빠름 / 한눈에 비교 / 모바일에서 즉시**
- 현재 단계: MVP 운영 중. 로그인·회원 없음. 누구나 들어오면 바로 검색.

**우리가 의도적으로 안 하는 것** (디자인에도 넣지 마세요):
- 회원가입/로그인, 마이페이지
- 합주실 상세페이지, 지도, 사진 갤러리, 커뮤니티/리뷰 작성
- 자체 결제 (예약은 외부 링크로만)
- "데이터가 N시간 전 거예요" 같은 신선도 표시 (내부적으로만 관리, 사용자에겐 숨김)
- 좌측 메뉴/탭 네비게이션/포털형 구조

## 2. 누가 쓰나 (사용자)

로그인 없는 익명 사용자. 크게 세 부류:
1. **급한 검색** — "오늘 저녁 비는 데 어디 없나" 빠르게 보고 바로 예약 링크로 이동
2. **밴드 멤버(재방문)** — 늘 비슷한 조건(같은 지역·시간)으로 자주 들어옴 → 마지막 검색 조건이 저장돼 있어야 함
3. **밴드 리더/총무** — 여러 날짜를 펼쳐놓고 인원·가격·방을 비교한 뒤 결정

**플랫폼: 모바일 우선.** 세로 휴대폰 화면(폭 약 430px)이 1순위. 데스크톱 대시보드가 아님. 한 손 엄지로 조작하는 걸 전제로.

## 3. [고정] 화면 구조 & 기능 (UX는 유지, 비주얼만 새로)

아래 화면과 기능, 정보 위계, 흐름은 **그대로 유지**합니다. 단, 각 요소를 "어떻게 보이게 할지"는 전부 새로 디자인하세요.

### 화면 A — 진입/지역 선택 (Open Screen)
- 브랜드(🎸 + "합주실닷컴") + 태그라인 "합주실 빈 시간, 한 곳에서"
- "어느 동네에서 찾으세요?" → 지역 빠른 선택 버튼들 + "전체 지역 둘러보기"
- 로딩 스켈레톤 / 로드 실패 시 "지역 목록을 불러오지 못했어요" + "다시 시도"
- *참고: 재방문(저장된 조건 있음) 시엔 이 화면을 건너뛰고 바로 결과 화면으로 감*

### 화면 B — 검색 결과 (메인, 가장 중요)
이 화면이 서비스의 99%입니다. 들어오면 **설명 없이 곧장 빈 시간 결과**가 보여야 합니다.

1. **상단 바**: "예약 가능 시간" 제목 + "HH:MM 업데이트" (마지막 수집 시각)
2. **필터 칩 줄** (가로 한 줄):
   - 시간대 칩: 오전 / 오후 / 저녁 / 밤 / 직접 범위 (기본 "아무 때나")
   - 날짜 칩: "일주일 내" / "N일 선택"
   - 지역 칩: "전체 지역" / 특정 지역 / "지역명 외 N"
   - 상세 필터 버튼(아이콘): 누르면 하단 시트 오픈
   - *각 칩을 탭하면 그 자리에서 빠른 팝오버가 뜸 (시간대 프리셋, 30일 달력, 지역 다중선택)*
3. **결과 리스트** (스크롤):
   - **날짜별로 그룹** (날짜 헤더 + "N곳" 카운트)
   - 날짜 안에 **합주실 카드** 나열. 카드 1개 = 합주실 1곳:
     - 아바타(이미지 또는 이니셜), 이름, 지역, 리뷰 키워드, **빈 시간 슬롯들**, 가격(예: "4.0–6.0만원/시간")
     - 빈 시간은 작은 알약 태그로 (예: `09:00`, `10:00~12:00`)
     - 메인 CTA: **"예약하기 ↗"** (외부 링크)
     - 펼치기: **"방 N개 · 방별로 보기"** → 방마다 시간/가격/정원 + "이 방 예약 ↗"
4. **상태들**:
   - 로딩: 스켈레톤(최초) / 상단 인디터미닛 진행바(갱신 중)
   - 에러: 상단 배너
   - 빈 결과: "조건에 맞는 빈 시간이 없어요" + 조건 완화 제안(날짜 초기화 / 인원 줄이기 / 시간 제한 해제)
   - 연속시간 부족: "이 날은 연속 N시간이 안 돼요"

### 화면 C — 상세 필터 (하단 바텀시트)
- 드래그로 내려서 닫는 바텀시트
- 섹션: **지역**(다중선택 칩) / **날짜**(달력) / **연속 시간**(1·2·3·4시간) / **인원**(1~10명 스테퍼)
- 하단 액션: "초기화" + "결과 N곳 보기"

### 핵심 인터랙션 원칙 [고정]
- **결과 우선** — 인트로/가입 없이 바로 결과
- **즉시 반응** — 필터를 바꾸면 (서버 재요청 없이) 그 자리에서 즉시 목록이 갱신됨. 칩/필터 조작에 로딩 스피너 안 돎.
- **핵심 흐름엔 모달 없음** — 검색·결과가 주 화면, 끊는 다이얼로그 금지 (바텀시트/팝오버만 허용)
- **밀도 > 여백 낭비** — 카드 그리드보다 한눈에 많이 보이는 콤팩트 리스트. 단, 답답하지 않게.
- **밖으로 링크** — 상세 예약은 외부에서. 우리는 비교까지만.

## 4. [고정] 브랜드 색 & 톤

- **주 색상(절대 고정): 틸/시안 계열.** 기준값 `oklch(0.60 0.16 200)` ≈ **#2EACC8**.
  파생: 진한 텍스트용 ≈ #0A7D8C, 아주 옅은 틴트 배경 ≈ #F0F9FB, 포커스 링 ≈ #7FD4E0.
  → 이 색상 계열(휴 200° 부근의 틸/시안)은 유지하세요. 채도/명암의 미세 조정은 가능하지만 다른 색상(보라·핑크·초록 등)으로 바꾸지 마세요.
- **무드: 깔끔하고 신뢰감 있는 토스(Toss)풍 중성 그레이 + 틸 포인트.**
  중성 그레이 텍스트: 진함 #191F28 / 보조 #4E5968 / 흐림 #6B7684, 면 #FFFFFF, 가라앉은 면 #F2F4F6, 보더 #DDE1E6.
  → **이 그레이/무드는 "출발점"일 뿐, 더 좋은 방향이 있으면 제안해도 됩니다.** 주 색상만 고정.
- 상태색: 주말/경고 = 따뜻한 오렌지·레드 계열, 강조 = 노랑. (역할만 유지, 톤 재해석 가능)
- 한글 가독성이 가장 중요. 현재 폰트는 Pretendard. **더 나은 한글 타이포 제안 환영** (단 가변·가독·웹 사용 가능 폰트).

## 5. [참고] 데이터/도메인 (디자인이 담아야 할 정보)

- **지역(Area)**: 홍대·강남 등
- **합주실(Studio)**: 이름, 지역, 이미지, 평점/리뷰수, 리뷰 키워드
- **방(Room)**: 이름, 시간당 가격(원), 정원(min~max인)
- **슬롯(Slot)**: 날짜 + 시작~종료 시간 + 가격 + 예약 링크(외부)
- **필터 상태**: 지역[] / 날짜[] (오늘~+29일, 30일) / 시간대[] / 연속시간(1~4) / 인원(1~10)
- 가격 표기 예: "4.0–6.0만원/시간", 정원 예: "4~8인", 슬롯 예: `10:00~12:00`

## 6. [고정] 한글 카피 (그대로 사용)

- 헤더: "예약 가능 시간" · 갱신: "HH:MM 업데이트"
- 시간대: 오전 / 오후 / 저녁 / 밤 / "아무 때나"
- 날짜 칩: "일주일 내" / "N일 선택" · 지역 칩: "전체 지역" / "지역명 외 N"
- CTA: "예약하기 ↗" · 방: "방 N개 · 방별로 보기" / "방별 시간 접기" / "이 방 예약 ↗"
- 연속시간: "1시간"~"4시간" · 인원: "N명"
- 필터 시트: "초기화" / "결과 N곳 보기"
- 빈 상태: "조건에 맞는 빈 시간이 없어요" / "이 날은 연속 N시간이 안 돼요"
- 진입: "합주실 빈 시간, 한 곳에서" / "어느 동네에서 찾으세요?" / "전체 지역 둘러보기"

## 7. 지금 디자인이 "허접해 보이는" 의심 지점 (이걸 풀어주세요)

현재는 CSS 토큰으로 토스풍을 흉내 냈지만 전체적으로 어설프게 느껴집니다. 새 디자인에서 특히 신경 써 주세요:

- **시각적 위계** — 합주실 이름 / 가격 / 시간 슬롯 / CTA 사이의 강약이 흐릿함. 무엇을 먼저 보게 할지 명확히.
- **리스트 카드의 완성도** — 아바타+이름+지역+시간 알약+가격+버튼이 한 카드 안에서 정렬·간격·정돈이 안 됨. 밀도 높으면서도 숨 쉬는 카드.
- **시간 슬롯 알약** — 핵심 정보인데 그냥 회색 태그라 밋밋함. 한눈에 "언제 비는지" 스캔되게.
- **필터 칩 줄 + 팝오버 + 바텀시트**의 일관된 시각 언어.
- **빈 상태/스켈레톤/에러**까지 완성도 있게 (대충 만든 티가 가장 잘 나는 곳).
- **틸 포인트 색을 절제 있게** — 남용하면 촌스럽고, 안 쓰면 밋밋함. 강조 위계를 설계.
- **전체 무드** — "급조한 MVP"가 아니라 "잘 만든 도구" 느낌. 모던하고 정돈된, 신뢰 가는 인상.
