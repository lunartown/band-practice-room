# DB 스키마

> 작성일: 2026-06-15
> 상태: 초안

## 1. 원칙

MVP는 ORM 없이 SQL을 직접 작성한다. 따라서 애플리케이션 코드보다 먼저 SQL 스키마를 명확히 둔다.

기본 방침:

- 시간대는 한국 시간 기준으로 해석한다
- 슬롯은 `slot_date + start_minute + end_minute`로 저장한다
- `24:00` 같은 종료 시각을 안정적으로 표현하기 위해 `time` 타입 대신 분 단위 정수를 쓴다
- `practice_rooms.id`, `rooms.id`는 API에도 노출되는 안정적인 text id를 쓴다
- 수집 실행/작업/슬롯 id는 DB 내부 식별자로 `bigserial`을 쓴다
- 가격은 원화 정수로 저장한다
- 가격 필터 기준은 시간당 가격이다
- JSON 응답은 DB row를 그대로 노출하지 않고 API 레이어에서 조립한다

## 3. practice_rooms

```sql
create table practice_rooms (
  id text primary key,
  name text not null,
  area text not null,
  area_tags text[] not null default '{}',
  address text,
  source_type source_type not null default 'naver',
  source_url text not null,
  booking_url text,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index idx_practice_rooms_area on practice_rooms (area);
create index idx_practice_rooms_area_tags on practice_rooms using gin (area_tags);
create index idx_practice_rooms_active on practice_rooms (is_active);
```

## 4. rooms

```sql
create table rooms (
  id text primary key,
  practice_room_id text not null references practice_rooms (id),
  name text not null,
  base_price_per_hour integer,
  base_price_source price_source not null default 'unknown',
  capacity_min integer,
  capacity_max integer,
  source_item_id text,
  source_item_url text,
  booking_url text,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint rooms_base_price_non_negative check (base_price_per_hour is null or base_price_per_hour >= 0),
  constraint rooms_capacity_valid check (
    capacity_min is null
    or capacity_max is null
    or capacity_min <= capacity_max
  )
);

create index idx_rooms_practice_room on rooms (practice_room_id);
create index idx_rooms_active on rooms (is_active);
create unique index idx_rooms_source_item
  on rooms (practice_room_id, source_item_id)
  where source_item_id is not null;
```

## 5. scrape_jobs

PostgreSQL job table이다. Worker는 `pending` 작업을 `scheduled_at` 순서로 가져가고, 동시 실행 시에는 `for update skip locked`로 충돌을 피한다.

```sql
create table scrape_jobs (
  id bigserial primary key,
  practice_room_id text not null references practice_rooms (id),
  target_date_from date not null,
  target_date_to date not null,
  status scrape_status not null default 'pending',
  attempts integer not null default 0,
  max_attempts integer not null default 3,
  scheduled_at timestamptz not null default now(),
  started_at timestamptz,
  finished_at timestamptz,
  locked_at timestamptz,
  locked_by text,
  error_message text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint scrape_jobs_date_range_valid check (target_date_from <= target_date_to),
  constraint scrape_jobs_attempts_valid check (attempts >= 0 and max_attempts > 0)
);

create index idx_scrape_jobs_pick
  on scrape_jobs (status, scheduled_at, id)
  where status = 'pending';

create index idx_scrape_jobs_practice_room_created
  on scrape_jobs (practice_room_id, created_at desc);
```

작업 선점 쿼리 형태:

```sql
with picked as (
  select id
  from scrape_jobs
  where status = 'pending'
    and scheduled_at <= now()
  order by scheduled_at asc, id asc
  for update skip locked
  limit 1
)
update scrape_jobs j
set
  status = 'running',
  attempts = attempts + 1,
  started_at = now(),
  locked_at = now(),
  locked_by = $1,
  updated_at = now()
from picked
where j.id = picked.id
returning j.*;
```

create table scrape_runs (

## 7. availability_slots

MVP 조회는 `available` 중심이다. 그래도 슬롯 상태 컬럼은 둔다. 나중에 unavailable 저장이나 변경 감지가 필요해질 때 테이블을 갈아엎지 않기 위해서다.

```sql
create table availability_slots (
  id bigserial primary key,
  room_id text not null references rooms (id),
  practice_room_id text not null references practice_rooms (id),
  scrape_run_id bigint references scrape_runs (id),
  slot_date date not null,
  start_minute smallint not null,
  end_minute smallint not null,
  status slot_status not null default 'available',
  price_per_hour integer,
  total_price integer,
  price_source price_source not null default 'unknown',
  scraped_at timestamptz not null,
  source_url text,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint availability_slots_minute_range check (
    start_minute >= 0
    and start_minute < 1440
    and end_minute > start_minute
    and end_minute <= 1440
  ),
  constraint availability_slots_price_non_negative check (
    (price_per_hour is null or price_per_hour >= 0)
    and (total_price is null or total_price >= 0)
  ),
  constraint availability_slots_room_unique unique (
    room_id,
    slot_date,
    start_minute,
    end_minute
  )
);

create index idx_availability_slots_lookup
  on availability_slots (slot_date, start_minute, end_minute)
  where status = 'available';

create index idx_availability_slots_room_date
  on availability_slots (room_id, slot_date);

create index idx_availability_slots_practice_room_date
  on availability_slots (practice_room_id, slot_date);

create index idx_availability_slots_scraped_at
  on availability_slots (scraped_at desc);
```

`practice_room_id`는 `rooms`를 통해 유도할 수 있지만, 조회 쿼리에서 지역/합주실 필터를 자주 쓰므로 의도적으로 중복 저장한다. 저장 시 애플리케이션에서 `room_id`와 일치하는 `practice_room_id`를 넣어야 한다.

## 8. manual_price_overrides

가격 수집이 누락되거나 원본 가격 표기가 불안정한 경우 운영자가 수기로 보정한다.

```sql
create table manual_price_overrides (
  id bigserial primary key,
  room_id text not null references rooms (id),
  price_per_hour integer not null,
  note text,
  starts_on date,
  ends_on date,
  is_active boolean not null default true,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),

  constraint manual_price_overrides_price_non_negative check (price_per_hour >= 0),
  constraint manual_price_overrides_date_range_valid check (
    starts_on is null
    or ends_on is null
    or starts_on <= ends_on
  )
);

create index idx_manual_price_overrides_room_active
  on manual_price_overrides (room_id, is_active);
```

## 9. 조회 쿼리 기준

### 9.1 30일 available 슬롯 조회

```sql
select
  s.id,
  s.room_id,
  s.practice_room_id,
  s.slot_date,
  s.start_minute,
  s.end_minute,
  coalesce(s.price_per_hour, r.base_price_per_hour) as price_per_hour,
  coalesce(
    s.total_price,
    round(
      coalesce(s.price_per_hour, r.base_price_per_hour)
      * ((s.end_minute - s.start_minute)::numeric / 60)
    )::integer
  ) as total_price,
  s.price_source,
  s.scraped_at,
  coalesce(s.source_url, r.booking_url, r.source_item_url, p.booking_url, p.source_url) as booking_url
from availability_slots s
join rooms r on r.id = s.room_id
join practice_rooms p on p.id = s.practice_room_id
where s.status = 'available'
  and s.slot_date between $1 and $2
  and p.is_active = true
  and r.is_active = true
  and ($3::text[] is null or p.area = any($3) or p.area_tags && $3)
  and ($4::text[] is null or p.id = any($4))
  and ($5::text[] is null or s.room_id = any($5))
  and ($6::integer is null or s.start_minute >= $6)
  and ($7::integer is null or s.end_minute <= $7)
  and ($8::integer is null or r.capacity_max is null or r.capacity_max >= $8)
  and ($9::integer is null or coalesce(s.price_per_hour, r.base_price_per_hour) <= $9)
  and ($10::boolean = true or s.scraped_at >= $11::timestamptz)
order by
  s.slot_date asc,
  s.start_minute asc,
  p.area asc,
  p.name asc,
  r.name asc;
```

### 9.2 수집 결과 저장

같은 방/날짜/시간 슬롯은 upsert한다.

```sql
insert into availability_slots (
  room_id,
  practice_room_id,
  scrape_run_id,
  slot_date,
  start_minute,
  end_minute,
  status,
  price_per_hour,
  total_price,
  price_source,
  scraped_at,
  source_url
)
values (
  $1, $2, $3, $4, $5, $6, 'available', $7, $8, $9, $10, $11
)
on conflict (room_id, slot_date, start_minute, end_minute)
do update set
  practice_room_id = excluded.practice_room_id,
  scrape_run_id = excluded.scrape_run_id,
  status = excluded.status,
  price_per_hour = excluded.price_per_hour,
  total_price = excluded.total_price,
  price_source = excluded.price_source,
  scraped_at = excluded.scraped_at,
  source_url = excluded.source_url,
  updated_at = now();
```

MVP에서는 수집한 날짜 범위 안에서 새 결과에 없는 기존 available 슬롯을 어떻게 처리할지 별도 정책이 필요하다. 기본 정책은 해당 `practice_room_id + target_date_from~to` 범위의 기존 슬롯을 `unknown`으로 바꾼 뒤 새 available 슬롯을 upsert하는 방식이다.
