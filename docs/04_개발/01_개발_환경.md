# 개발 환경

> 작성일: 2026-06-13 (갱신: 2026-06-22)
> 상태: 작성 — 실제 구조(`src/`) 기준

## 1. 요구 버전

- Node.js 22 (GitHub Actions 워크플로우 기준)
- Docker / Docker Compose (로컬 PostgreSQL용)

## 2. 디렉터리 구조

코드는 `src/` 아래 세 패키지로 나뉜다.

- `src/api` — NestJS API 서버 (ESM, `tsx` 실행)
- `src/web` — Vite + React 프론트엔드 (Capacitor로 iOS·Android 패키징)
- `src/scraper` — 네이버·스페이스클라우드 예약 가능 시간 수집 워커

## 3. 로컬 PostgreSQL

로컬 개발 DB는 Docker Compose로 실행한다.

```sh
docker compose up -d postgres
```

접속 정보:

- Host: `localhost`
- Port: `15432`
- Database: `band_practice_room`
- User: `band_practice_room`
- Password: `band_practice_room`
- URL: `postgres://band_practice_room:band_practice_room@localhost:15432/band_practice_room`

`DATABASE_URL` 환경변수로 위 URL을 사용한다.

## 4. API 실행

```sh
cd src/api
npm install
npm run db:migrate   # 마이그레이션 (idempotent)
npm run db:seed      # 합주실 시드 + 스크랩 매핑 (slots TRUNCATE 후 재생성)
npm run dev          # tsx watch
```

테스트:

```sh
cd src/api
npm test             # jest --runInBand
```

## 5. 스크래퍼 실행

```sh
cd src/scraper
npm install
npm start            # 1회 수집 (RUN_ONCE 기본 동작)
```

`DATABASE_URL`이 가리키는 DB의 `slots`를 네이버·스페이스클라우드에서 갱신한다. 정기 수집은 GitHub Actions(`.github/workflows/scrape.yml`)가 매시 정각에 수행한다.

## 6. 웹 실행

```sh
cd src/web
npm install
npm run dev          # vite --host 0.0.0.0
```

기본값은 mock API다. 실제 API로 붙이려면:

```sh
VITE_USE_MOCK_API=false VITE_API_BASE_URL=/api/v1 npm run dev
```

개발 서버의 `/api`는 기본적으로 Render 실 API로 프록시된다. 휴대폰에서 Mac IP(`http://192.168.x.x:5173/`)로 로컬 웹을 확인할 때도 CORS 없이 실데이터를 볼 수 있다.

> **프런트 변경은 여기서(로컬) 검증한다.** Vercel 프리뷰는 무료 플랜 rate limit 때문에 `dev` 브랜치에서만 띄운다. 기능 브랜치는 Vercel에 올리지 말고 위처럼 로컬 + 라이브 백엔드 프록시로 확인한 뒤 `dev`로 PR한다. 정책 근거는 [03_배포_전략.md](03_배포_전략.md) "프리뷰 배포 필터" 참고.

로컬 API를 직접 확인해야 할 때만 proxy 타깃을 덮어쓴다:

```sh
VITE_USE_MOCK_API=false VITE_API_BASE_URL=/api/v1 \
VITE_DEV_API_PROXY_TARGET=http://127.0.0.1:3000 \
npm run dev
```

자세한 내용은 `src/web/README.md` 참고.
