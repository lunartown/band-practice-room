## 3. MVP 범위

MVP는 **합주실 예약 가능 시간 검색**에 집중한다.

포함:

- 서울 주요 합주실 일부
- 최소 30일 범위의 예약 가능 시간 조회
- 시간/합주실 조건 필터
- 원본 예약 페이지 링크
- 예약 가능 데이터 사전 수집
- 수집 실패 로그 (내부 운영용)

제외:

- 직접 예약
- 직접 결제
- 커뮤니티
- 공연 정보
- 악보 검색
- 밴드 활동 가이드
- 업체 소개 중심 정보 페이지
- 공연장 예약 전체 기능
- 개인연습실 또는 개인 연습 상품 검색

## 4. UX 요구사항

### UX-001. 첫 화면은 검색 결과 중심

첫 화면의 중앙 핵심 공간은 안내문, 공지, 서비스 설명이 아니라 날짜/시간 조건과 예약 가능 결과가 차지해야 한다.

### UX-002. 설명 텍스트 최소화

“선택한 시간 기준으로...” 같은 시스템 설명은 보조 위치로 내려야 한다. 사용자는 설명을 읽으러 온 것이 아니라 빈 시간을 보러 온다.

### UX-003. 진입 방해 요소 금지

공지사항 모달이 핵심 화면을 가리면 안 된다. 공지는 별도 영역이나 낮은 위계로 둔다.

### UX-004. 조건 변경 즉시 반영

조건 검색마다 10초 이상 기다리는 구조는 제품의 본질과 충돌한다. 프론트는 필요한 데이터 묶음을 미리 받아 클라이언트에서 즉시 필터링해야 한다.

## 5. 데이터 요구사항

- 예약 가능 시간은 `date`, `room`, `start_time`, `end_time`, `status`, `scraped_at`을 가져야 한다
- 합주실 정보는 최소한의 식별 정보와 원본 예약 URL만 우선 저장한다
- 가격은 MVP 결과에 포함한다
- 가격이 원본에서 수집되지 않으면 운영자가 수기 입력할 수 있어야 한다
- 가격 출처는 수집값인지 수기 입력값인지 구분한다
- 지역 정보는 필터 가치가 있을 때 추가한다
- 업체 소개, 상세 장비, 사진 등은 MVP 데이터 모델의 중심이 아니다
