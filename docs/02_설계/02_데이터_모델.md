# 데이터 모델

> 작성일: 2026-06-15 (구현 반영 갱신: 2026-06-22)
> 상태: 구현 반영 — 실제 `src/api/db` 마이그레이션 기준

## 1. 기준

데이터 모델은 [도메인 모델](05_도메인_모델.md)의 Figma 기준 엔티티에서 출발했다.

§3은 초기 스키마(`001_init.sql`)의 핵심 테이블이고, §6은 이후 마이그레이션(`002`~`006`)으로 추가된 작업 큐·합주실 비주얼·리뷰 키워드·스페이스클라우드 소스를 정리한다. 실제 컬럼은 `src/api/db/migrations`가 source of truth다.

## 2. 표기 규칙

- 테이블명: snake_case 복수형
- 컬럼명: snake_case
- ID: `BIGSERIAL`
- 날짜: `DATE`
- 시간: `TIME`
- 금액: `INTEGER`, 원(KRW) 단위, 모르는 경우 `NULL`
- enum 성격 값은 `VARCHAR`와 `CHECK` 제약으로 표현한다

`CHECK` 제약은 컬럼에 허용된 값만 들어오도록 DB가 검증하는 규칙이다. 예를 들어 `slots.status`에 `AVAILABLE`, `UNAVAILABLE`, `UNKNOWN` 외 값이 저장되는 것을 막는다.

## 3. 테이블 정의

### areas

```sql
CREATE TABLE areas (
  id BIGSERIAL PRIMARY KEY,
  slug VARCHAR(64) NOT NULL UNIQUE,
  name VARCHAR(64) NOT NULL,
  "order" SMALLINT NOT NULL,
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studios

```sql
CREATE TABLE studios (
  id BIGSERIAL PRIMARY KEY,
  slug VARCHAR(128) UNIQUE,
  name VARCHAR(128) NOT NULL,
  description TEXT,
  primary_area_id BIGINT REFERENCES areas(id),
  address VARCHAR(256),
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studio_areas

```sql
CREATE TABLE studio_areas (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  area_id BIGINT NOT NULL REFERENCES areas(id),
  UNIQUE (studio_id, area_id)
);
```

### sources

```sql
CREATE TABLE sources (
  id BIGSERIAL PRIMARY KEY,
  name VARCHAR(128) NOT NULL,
  url VARCHAR(512) NOT NULL,
  auth_kind VARCHAR(32) CHECK (auth_kind IN ('NONE', 'FORM_LOGIN', 'COOKIE', 'MANUAL_SESSION')),
  credential_key VARCHAR(256),
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studio_sources

```sql
CREATE TABLE studio_sources (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  source_id BIGINT NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url VARCHAR(512),
  UNIQUE (studio_id, source_id)
);
```

### rooms

```sql
CREATE TABLE rooms (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  name VARCHAR(128) NOT NULL,
  price_per_hour INTEGER,
  price_source VARCHAR(32) CHECK (price_source IN ('SCRAPED', 'MANUAL', 'UNKNOWN')),
  capacity_min SMALLINT,
  capacity_max SMALLINT,
  is_active BOOLEAN NOT NULL DEFAULT true,
  CHECK (capacity_min IS NULL OR capacity_max IS NULL OR capacity_max >= capacity_min)
);
```

### room_sources

```sql
CREATE TABLE room_sources (
  id BIGSERIAL PRIMARY KEY,
  room_id BIGINT NOT NULL REFERENCES rooms(id),
  source_id BIGINT NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url VARCHAR(512),
  UNIQUE (room_id, source_id)
);
```

### slots

```sql
CREATE TABLE slots (
  id BIGSERIAL PRIMARY KEY,
  room_id BIGINT NOT NULL REFERENCES rooms(id),
  date DATE NOT NULL,
  start_time TIME NOT NULL,
  end_time TIME NOT NULL,
  status VARCHAR(16) NOT NULL CHECK (status IN ('AVAILABLE', 'UNAVAILABLE', 'UNKNOWN')),
  price INTEGER,
  price_source VARCHAR(32) NOT NULL CHECK (price_source IN ('SCRAPED', 'MANUAL', 'UNKNOWN')),
  scraped_at TIMESTAMPTZ NOT NULL,
  CHECK (end_time > start_time),
  UNIQUE (room_id, date, start_time)
);
```

> `end_time > start_time` 제약은 `003_relax_slot_check.sql`에서 `end_time > start_time OR end_time = '00:00:00'`으로 완화했다. 자정(`00:00`)은 다음날 00:00를 의미하기 때문이다.

## 4. 인덱스 후보

```sql
CREATE INDEX idx_slots_date_status ON slots (date, status);
CREATE INDEX idx_slots_room_date ON slots (room_id, date);
CREATE INDEX idx_studios_primary_area ON studios (primary_area_id);
CREATE INDEX idx_studio_areas_area ON studio_areas (area_id, studio_id);
CREATE INDEX idx_rooms_studio_active ON rooms (studio_id, is_active);
```

## 5. 설계 메모

- `studios.primary_area_id`는 합주실의 대표 지역이다.
- `studio_areas`는 합주실이 여러 지역에 노출될 수 있을 때 사용한다. 대표 지역도 `studio_areas`에 함께 넣어두면 지역 필터 쿼리를 단순하게 유지할 수 있다.
- `slots.price`는 원본에서 가격을 알 수 없거나 방 기본가만 있는 경우를 고려해 `NULL`을 허용한다.
- `slots.status`는 도메인 모델 기준 `AVAILABLE`, `UNAVAILABLE`, `UNKNOWN` 값을 사용한다.
- 도메인 모델의 `Room.priceSource` 예시는 `UNKOWN`으로 표기되어 있으나, DB 값은 오탈자를 정규화해 `UNKNOWN`을 사용한다.
- `slots.price_source = UNKNOWN`이면 `slots.price`는 `NULL`일 수 있다.
- `slots.scraped_at`은 1차 백엔드에서는 seed 데이터가 만들어진 시각을 넣고, 스크래핑 도입 후에는 원본 슬롯 상태를 마지막으로 확인한 시각을 넣는다.
- `sources`, `studio_sources`, `room_sources`는 스크래핑 추적이 아니라 원본 예약 페이지 연결을 위한 기본 데이터다.

## 6. 이후 마이그레이션 (002~006)

초기 스키마(§3) 이후 추가된 테이블·컬럼. 실제 정의는 `src/api/db/migrations`에 있다.

### 6.1 작업 큐·수집 이력 (`002_scrape_jobs.sql`)

스크래퍼 도입과 함께 작업 큐와 수집 실행 이력을 추가했다. 도메인 모델 §12.2의 `ScrapeJob`/`ScrapeRun` 후보가 실제 테이블로 구현된 것이다.

- `scrape_jobs` — 수집 작업 큐. `studio_source_id`, `date_from/to`, `status('PENDING'|'RUNNING'|'SUCCESS'|'FAILED')`, `priority`, `run_after`(다음 실행 가능 시각), `attempts`, `last_error`. GitHub Actions cron이 이 큐를 한 바퀴 비운다.
- `scrape_runs` — 수집 실행 기록. `job_id`, `studio_id`, `source_id`, `status('SUCCESS'|'FAILED'|'PARTIAL')`, `rooms_found`, `slots_found`, `error_kind`, `error_message`.

### 6.2 합주실 비주얼 (`004_studio_visuals.sql`)

`studios`에 비주얼·평판 컬럼 추가.

- `image_url_manual` / `image_url_scraped` — 썸네일. 응답은 `COALESCE(manual, scraped)`로 매뉴얼 우선.
- `rating NUMERIC(2,1)` (0~5), `review_count INTEGER` — 시드로 수동 입력. (이전 단일 `image_url` 컬럼은 제거.)

### 6.3 리뷰 키워드 (`005_studio_review_keywords.sql`)

- `studios.review_keywords JSONB` — 네이버 키워드 리뷰 상위 N개. 형식: `[{"keyword": "시설이 깔끔해요", "count": 69}, ...]` (count 내림차순). 네이버 리뷰는 별점이 아니라 키워드식이라 이쪽을 품질 신호로 쓴다.

### 6.4 멀티 예약채널 (`006_spacecloud_source.sql`)

- `sources.code VARCHAR(32)` — 스크래퍼·예약링크 빌더가 소스 종류를 안정적으로 분기하는 키(`'naver'`, `'spacecloud'`). 이름(`name`)은 표시용.
- 스페이스클라우드 소스(`id=2`) 등록. 가용성 API 토큰은 `credential_key='SPACECLOUD_API_TOKEN'`으로 env 키를 가리킨다.
