# 데이터 모델

> 작성일: 2026-06-15
> 상태: 초안

## 1. 기준

[도메인 모델](05_도메인_모델.md)의 Figma 기준 엔티티와 스크래핑 MVP 보강 엔티티를 기준으로 작성한다.

- 테이블명: snake_case 복수형
- 컬럼명: snake_case
- 날짜: `DATE`
- 시각: `TIMESTAMPTZ`
- 시간: `TIME` (도메인 모델의 HHmm number 대신 PostgreSQL TIME 사용, 앱 레이어에서 변환)
- 금액: `INTEGER` (원 단위)
- enum: `VARCHAR` + `CHECK` 제약

## 2. 테이블 정의

### areas

```sql
CREATE TABLE areas (
  id        BIGSERIAL   PRIMARY KEY,
  slug      VARCHAR(64) NOT NULL UNIQUE,
  name      VARCHAR(64) NOT NULL,
  "order"   SMALLINT    NOT NULL,
  is_active BOOLEAN     NOT NULL DEFAULT true
);
```

### studios

```sql
CREATE TABLE studios (
  id          BIGSERIAL    PRIMARY KEY,
  slug        VARCHAR(128) UNIQUE,
  name        VARCHAR(128) NOT NULL,
  description TEXT,
  area_id     BIGINT       REFERENCES areas(id),
  address     VARCHAR(256),
  is_active   BOOLEAN      NOT NULL DEFAULT true
);
```

### studio_areas

```sql
CREATE TABLE studio_areas (
  id        BIGSERIAL PRIMARY KEY,
  studio_id BIGINT    NOT NULL REFERENCES studios(id),
  area_id   BIGINT    NOT NULL REFERENCES areas(id),
  UNIQUE (studio_id, area_id)
);
```

### sources

```sql
CREATE TABLE sources (
  id             BIGSERIAL    PRIMARY KEY,
  name           VARCHAR(128) NOT NULL,
  url            VARCHAR(512) NOT NULL,
  auth_kind      VARCHAR(32)  CHECK (auth_kind IN ('NONE', 'FORM_LOGIN', 'COOKIE', 'MANUAL_SESSION')),
  credential_key VARCHAR(256),
  is_active      BOOLEAN      NOT NULL DEFAULT true
);
```

### studio_sources

```sql
CREATE TABLE studio_sources (
  id           BIGSERIAL    PRIMARY KEY,
  studio_id    BIGINT       NOT NULL REFERENCES studios(id),
  source_id    BIGINT       NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url          VARCHAR(512),
  UNIQUE (studio_id, source_id)
);
```

### rooms

```sql
CREATE TABLE rooms (
  id             BIGSERIAL    PRIMARY KEY,
  studio_id      BIGINT       NOT NULL REFERENCES studios(id),
  name           VARCHAR(128) NOT NULL,
  price_per_hour INTEGER,
  price_source   VARCHAR(32)  CHECK (price_source IN ('SCRAPED', 'MANUAL', 'UNKNOWN')),
  capacity_min   SMALLINT,
  capacity_max   SMALLINT,
  is_active      BOOLEAN      NOT NULL DEFAULT true
);
```

### room_sources

```sql
CREATE TABLE room_sources (
  id           BIGSERIAL    PRIMARY KEY,
  room_id      BIGINT       NOT NULL REFERENCES rooms(id),
  source_id    BIGINT       NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url          VARCHAR(512),
  UNIQUE (room_id, source_id)
);
```

### scrape_jobs

```sql
CREATE TABLE scrape_jobs (
  id               BIGSERIAL   PRIMARY KEY,
  studio_source_id BIGINT      NOT NULL REFERENCES studio_sources(id),
  date_from        DATE        NOT NULL,
  date_to          DATE        NOT NULL,
  status           VARCHAR(16) NOT NULL DEFAULT 'PENDING'
                                CHECK (status IN ('PENDING', 'RUNNING', 'SUCCESS', 'FAILED')),
  priority         SMALLINT    NOT NULL DEFAULT 0,
  run_after        TIMESTAMPTZ,
  attempts         SMALLINT    NOT NULL DEFAULT 0,
  max_attempts     SMALLINT    NOT NULL DEFAULT 3,
  locked_at        TIMESTAMPTZ,
  last_error       TEXT,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
  CHECK (date_to >= date_from)
);
```

### scrape_runs

```sql
CREATE TABLE scrape_runs (
  id            BIGSERIAL   PRIMARY KEY,
  job_id        BIGINT      REFERENCES scrape_jobs(id),
  studio_id     BIGINT      NOT NULL REFERENCES studios(id),
  source_id     BIGINT      NOT NULL REFERENCES sources(id),
  date_from     DATE        NOT NULL,
  date_to       DATE        NOT NULL,
  status        VARCHAR(16) NOT NULL CHECK (status IN ('SUCCESS', 'FAILED', 'PARTIAL')),
  started_at    TIMESTAMPTZ NOT NULL,
  finished_at   TIMESTAMPTZ,
  rooms_found   INTEGER,
  slots_found   INTEGER,
  error_kind    VARCHAR(32) CHECK (error_kind IN ('TIMEOUT', 'AUTH_FAILED', 'PARSE_FAILED', 'NETWORK_ERROR', 'UNKNOWN')),
  error_message TEXT,
  CHECK (date_to >= date_from)
);
```

### slots

```sql
CREATE TABLE slots (
  id             BIGSERIAL   PRIMARY KEY,
  room_id        BIGINT      NOT NULL REFERENCES rooms(id),
  room_source_id BIGINT      REFERENCES room_sources(id),
  scrape_run_id  BIGINT      REFERENCES scrape_runs(id),
  date           DATE        NOT NULL,
  start_time     TIME        NOT NULL,
  end_time       TIME        NOT NULL,
  status         VARCHAR(16) NOT NULL CHECK (status IN ('AVAILABLE', 'UNAVAILABLE', 'UNKNOWN')),
  price          INTEGER,
  price_source   VARCHAR(32) NOT NULL DEFAULT 'UNKNOWN'
                                CHECK (price_source IN ('SCRAPED', 'MANUAL', 'ROOM_DEFAULT', 'UNKNOWN')),
  scraped_at     TIMESTAMPTZ NOT NULL,
  external_key   VARCHAR(256),
  booking_url    VARCHAR(512),
  UNIQUE (room_id, date, start_time)
);
```

## 3. 인덱스

```sql
-- 핵심 조회: 날짜 범위 기준 슬롯
CREATE INDEX idx_slots_room_date ON slots (room_id, date);
CREATE INDEX idx_slots_date_status ON slots (date, status);
CREATE INDEX idx_slots_scraped_at ON slots (scraped_at);

-- 합주실/방 필터
CREATE INDEX idx_studios_is_active ON studios (is_active);
CREATE INDEX idx_rooms_studio_active ON rooms (studio_id, is_active);

-- 수집 작업/실행 조회
CREATE INDEX idx_scrape_jobs_runnable ON scrape_jobs (status, run_after, priority);
CREATE INDEX idx_scrape_runs_target_started ON scrape_runs (studio_id, source_id, started_at DESC);
```

## 4. 설계 메모

- `studios.area_id`와 `studio_areas` 중복: `area_id`는 단일 지역 합주실의 빠른 필터용. `studio_areas`는 복수 지역 매핑용. 두 개를 병행해 사용한다.
- 도메인 모델에서 `Studio.isActive`는 `string`, `Area.isActive`는 `number`로 표기되어 있으나 DB에서는 모두 `BOOLEAN`으로 정의한다.
- 도메인 모델의 `Room.priceSource` enum 값 `UNKOWN`은 오탈자로 보여 DB에서는 `UNKNOWN`으로 정의한다.
- `slots`의 유니크 제약 `(room_id, date, start_time)`은 수집 재실행 시 upsert 기준이다.
- `slots.price`는 원본에서 가격을 수집하지 못하거나 방 기본가만 있는 경우를 고려해 nullable로 둔다.
- `slots.booking_url`이 null이면 API는 `room_sources.url`을 fallback으로 사용할 수 있다.
- `scrape_jobs`는 초기 큐 구현을 PostgreSQL만으로 시작하기 위한 작업 테이블이다.
- `scrape_runs`는 수집 실패가 전체 조회 실패로 이어지지 않도록 실패 범위와 원인을 남기는 이력 테이블이다.
