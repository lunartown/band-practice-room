# 데이터 모델

> 작성일: 2026-06-15
> 상태: 초안

## 1. 기준

데이터 모델은 [도메인 모델](05_도메인_모델.md)의 Figma 기준 엔티티를 기준으로 작성한다.

현재 단계는 기본 앱 껍데기를 만들기 위한 최소 스키마 초안이다. 스크래핑 실행 이력, 작업 큐, 실패 추적 테이블은 실제 스크래핑 기능을 붙일 때 별도 설계한다.

## 2. 표기 규칙

- 테이블명: snake_case 복수형
- 컬럼명: snake_case
- ID: `BIGSERIAL`
- 날짜: `DATE`
- 시간: `TIME`
- 금액: `INTEGER`, 원(KRW) 단위, 모르는 경우 `NULL`
- enum 성격 값은 우선 `VARCHAR`로 둔다

## 3. 테이블 정의

### areas

```sql
CREATE TABLE areas (
  id BIGSERIAL PRIMARY KEY,
  slug VARCHAR(64) NOT NULL UNIQUE,
  name VARCHAR(64) NOT NULL,
  "order" SMALLINT NOT NULL,
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studios

```sql
CREATE TABLE studios (
  id BIGSERIAL PRIMARY KEY,
  slug VARCHAR(128) UNIQUE,
  name VARCHAR(128) NOT NULL,
  description TEXT,
  primary_area_id BIGINT REFERENCES areas(id),
  address VARCHAR(256),
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studio_areas

```sql
CREATE TABLE studio_areas (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  area_id BIGINT NOT NULL REFERENCES areas(id),
  UNIQUE (studio_id, area_id)
);
```

### sources

```sql
CREATE TABLE sources (
  id BIGSERIAL PRIMARY KEY,
  name VARCHAR(128) NOT NULL,
  url VARCHAR(512) NOT NULL,
  auth_kind VARCHAR(32),
  credential_key VARCHAR(256),
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### studio_sources

```sql
CREATE TABLE studio_sources (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  source_id BIGINT NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url VARCHAR(512),
  UNIQUE (studio_id, source_id)
);
```

### rooms

```sql
CREATE TABLE rooms (
  id BIGSERIAL PRIMARY KEY,
  studio_id BIGINT NOT NULL REFERENCES studios(id),
  name VARCHAR(128) NOT NULL,
  price_per_hour INTEGER,
  price_source VARCHAR(32),
  capacity_min SMALLINT,
  capacity_max SMALLINT,
  is_active BOOLEAN NOT NULL DEFAULT true
);
```

### room_sources

```sql
CREATE TABLE room_sources (
  id BIGSERIAL PRIMARY KEY,
  room_id BIGINT NOT NULL REFERENCES rooms(id),
  source_id BIGINT NOT NULL REFERENCES sources(id),
  external_key VARCHAR(256),
  url VARCHAR(512),
  UNIQUE (room_id, source_id)
);
```

### slots

```sql
CREATE TABLE slots (
  id BIGSERIAL PRIMARY KEY,
  room_id BIGINT NOT NULL REFERENCES rooms(id),
  date DATE NOT NULL,
  start_time TIME NOT NULL,
  end_time TIME NOT NULL,
  status VARCHAR(16) NOT NULL,
  price INTEGER,
  price_source VARCHAR(32) NOT NULL,
  UNIQUE (room_id, date, start_time)
);
```

## 4. 인덱스 후보

```sql
CREATE INDEX idx_slots_date_status ON slots (date, status);
CREATE INDEX idx_slots_room_date ON slots (room_id, date);
CREATE INDEX idx_studios_primary_area ON studios (primary_area_id);
CREATE INDEX idx_rooms_studio_active ON rooms (studio_id, is_active);
```

## 5. 설계 메모

- `studios.primary_area_id`는 합주실의 대표 생활권이다.
- `studio_areas`는 합주실이 여러 생활권에 노출될 수 있을 때 사용한다.
- `slots.price`는 원본에서 가격을 알 수 없거나 방 기본가만 있는 경우를 고려해 `NULL`을 허용한다.
- `slots.status`는 도메인 모델 기준 `AVAILABLE`, `UNAVAILABLE`, `UNKNOWN` 값을 사용한다.
- `CHECK` 제약은 실제 DB 마이그레이션을 작성할 때 enum 값이 확정되면 추가한다.
