# 정보 구조

> 작성일: 2026-06-15
> 상태: 초안

## 1. 설계 전제

이 서비스의 중심 정보는 합주실 소개가 아니라 **예약 가능 시간**이다.

사용자는 “어떤 합주실인가?”보다 먼저 “내가 원하는 날짜와 시간에 비는 방이 있는가?”를 알고 싶다. 따라서 정보 구조는 업체 상세 페이지 중심이 아니라 조건 검색 결과 중심으로 잡는다.

## 2. 주요 정보 단위

| 정보 단위 | 설명 | 사용자에게 보이는 방식 |
| --- | --- | --- |
| PracticeRoom | 합주실 사업장 | 합주실 이름, 지역, 원본 예약 링크 |
| Room | 합주실 내부 방 | 방 이름, 가격 또는 기준 요금, 예약 가능 시간 |
| AvailabilitySlot | 특정 날짜/시간의 예약 가능 상태 | 날짜, 시작 시간, 종료 시간, 가능/불가 |
| Source | 네이버 예약 등 원본 예약 페이지 | 원본 예약 페이지로 이동 |
| ScrapeRun | 수집 실행 이력 | 사용자에게 직접 노출하지 않음 |
| Freshness | 데이터 갱신 상태 (내부 운영 지표) | 사용자에게 노출하지 않음. 재수집 주기 판단에만 사용 |

## 3. 사용자 관점 정보 위계

사용자 화면에서 우선순위는 아래 순서다.

1. 날짜 범위
2. 시간대
3. 예약 가능한 방
4. 합주실 이름과 지역
5. 원본 예약 페이지 이동
6. 가격 또는 기준 요금
7. 보조 정보

합주실 소개, 장비 설명, 사진, 상세 위치 안내는 MVP 중심 정보가 아니다. 필요하다면 원본 예약 페이지 또는 네이버 지도에서 확인하게 한다.

## 4. 탐색 구조

```txt
예약 가능 시간 검색
  ├─ 날짜 범위 선택
  ├─ 시간대 선택
  ├─ 지역/합주실 필터
  └─ 결과 목록
      ├─ 날짜별 그룹
      ├─ 합주실별 그룹
      └─ 방별 예약 가능 슬롯
          └─ 원본 예약 페이지 이동
```

## 5. 기본 화면 구조

### 5.1 검색 화면

서비스의 첫 화면이다. 별도 홈/랜딩/공지 화면을 앞에 두지 않는다.

포함:

- 날짜 범위 선택
- 시간대 선택
- 지역/합주실 필터
- 예약 가능 결과

제외:

- 진입 모달
- 긴 안내 텍스트
- 업체 소개 카드
- 커뮤니티/공연장/개인연습실 메뉴

### 5.2 결과 항목

결과 항목은 “합주실 카드”보다 “예약 가능 슬롯”을 중심으로 구성한다.

```txt
날짜
  시간대
    합주실 / 방 / 가격 / 원본 예약 링크
```

기본 결과 그룹은 날짜/시간 기준으로 고정한다. 사용자는 보통 특정 합주실보다 “언제 가능한가”를 먼저 찾기 때문이다.

합주실 기준 보기는 후속 보조 뷰로 둘 수 있지만, MVP 첫 화면의 기본 구조는 아니다.

## 6. 날짜 범위 구조

MVP는 오늘/내일이 아니라 **최소 30일 범위**를 기준으로 한다.

기본 진입 시:

- 오늘부터 30일까지의 데이터를 조회한다
- 가까운 날짜를 먼저 보여준다
- 사용자는 날짜 범위를 넓히거나 좁힐 수 있다

MVP에서는 30일 전체의 **예약 가능 슬롯**을 한 번에 내려준다. 한국 합주실 시장 규모와 초기 수집 대상 수를 고려하면 응답 크기가 제품 방향을 꺾을 정도로 크지 않을 가능성이 높다.

```txt
API
  → 30일 범위 메타데이터 + available slots 반환
Client
  → 날짜/시간/지역/합주실 필터 즉시 처리
```

응답이 실제 운영에서 커질 경우에만 날짜 단위 또는 주 단위 chunk API를 추가한다. 초기 설계의 기본값은 일괄 응답이다.

## 7. 상태 정보 (내부 운영 전용)

데이터 정확도는 제품이 책임지고 보장한다. 단, **신선도/stale 상태는 사용자 화면에 노출하지 않는다.** 사용자는 데이터가 신선한지를 판단할 필요가 없고, 당연히 신선하다고 믿고 쓴다 (의사결정 로그 2026-06-21).

아래 상태값은 worker가 **재수집 주기와 실패 처리를 판단하기 위한 내부 지표**로만 사용한다.

| 상태 | 의미 | 내부 처리 |
| --- | --- | --- |
| fresh | 최근 수집 성공 | 정상 노출 |
| stale | 수집 후 시간이 지남 | 재수집 우선순위를 높임 (사용자에게는 구분 없이 노출) |
| failed | 최근 수집 실패 | 해당 합주실/슬롯을 결과에서 조용히 제외, 재시도 큐잉 |
| unknown | 수집 전 | 결과에 포함하지 않음 |

핵심: 신선도는 **사용자에게 떠넘길 정보가 아니라 제품이 내부적으로 흡수할 운영 책임**이다.

## 8. 지역 구조

지역 필터는 행정구가 아니라 지역 기준으로 한다.

예:

- 홍대
- 합정
- 신촌
- 강남
- 건대

사용자는 보통 “마포구 합주실”보다 “홍대 합주실”, “합정 합주실”처럼 지역으로 탐색한다.
