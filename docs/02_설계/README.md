## 설계 전제

- 제품 본질은 합주실 정보 포털이 아니라 예약 가능 시간 검색 엔진이다.
- 최종 예약은 외부 원본 페이지에서 일어나는 구조를 인정한다.
- 그 대신 내부 탐색은 사전 적재와 클라이언트 필터링으로 빠르게 만든다.
- 데이터 신선도와 수집 실패 처리는 기능이 아니라 신뢰도 설계의 일부다.

## 확정된 설계 결정

- MVP는 30일 전체 available 슬롯을 한 응답으로 내려준다
- 응답은 `practiceRooms`, `slots`, `freshness`로 분리해 중복을 줄인다
- 프론트엔드는 Vite SPA + React로 시작한다
- 스케줄러는 API 내부 cron이 아니라 별도 worker process로 설계한다
- 초기 작업 큐는 PostgreSQL job table로 구현한다
- 기본 결과 그룹은 날짜/시간 기준으로 한다
- 가격은 MVP 결과에 포함하고, 수집 누락 시 수기 입력값을 사용할 수 있게 한다
- 지역 필터는 행정구가 아니라 지역 기준으로 한다
- 슬롯 시간은 날짜와 하루 안의 분 단위 범위로 표현한다

## 현재 논의가 필요한 지점

- stale 데이터 표시 기준을 몇 시간으로 둘지
- 지역 경계가 애매한 합주실을 여러 지역에 중복 노출할지
