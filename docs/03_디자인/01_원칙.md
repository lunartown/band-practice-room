# 디자인 원칙

> 작성일: 2026-06-13
> 상태: 초안

## 1. 제품 경험 원칙

### 원칙 1. 첫 화면은 결과여야 한다

사용자가 보고 싶은 것은 서비스 설명이 아니라 “지금 비는 방”이다. 첫 화면의 가장 중요한 공간은 날짜, 시간 조건, 예약 가능 결과가 차지해야 한다.

### 원칙 2. 설명하지 말고 보여준다

“선택한 시간 기준으로...” 같은 시스템 설명은 핵심 공간을 차지하면 안 된다. 사용자가 조작하면 결과가 바로 바뀌어야 하고, 설명은 보조 위치에 둔다.

### 원칙 3. 모달로 진입을 막지 않는다

공지사항, 안내, 이벤트, 업데이트 정보가 메인 검색 경험을 가리면 안 된다. 중요한 공지도 낮은 위계의 배너나 별도 영역으로 처리한다.

### 원칙 4. 모바일을 기준으로 판단한다

합주실 예약 탐색은 단톡방, 이동 중, 급한 상황에서 일어날 가능성이 높다. PC 대시보드형 사이드바 구조보다 모바일에서 즉시 조건을 바꾸고 결과를 보는 흐름이 우선이다.

### 원칙 5. 정보 포털처럼 보이면 실패다

합주실 소개, 지도, 장비 설명, 게시판, 가이드가 중심이 되면 네이버 지도와 비교해 이길 이유가 약하다. 이 서비스는 정보량이 아니라 빈 시간 비교 속도로 차별화한다.

### 원칙 6. MVP는 합주실만 다룬다

공연장 예약, 개인연습실, 커뮤니티는 같은 화면 안에 섞지 않는다. MVP 화면은 합주실 예약 가능 시간 검색 하나에만 집중한다.

### 원칙 7. 날짜는 넓게 보되 조작은 가볍게 만든다

오늘/내일 바로가기만으로는 부족하다. 최소 30일 범위를 탐색할 수 있어야 하지만, 모바일에서 날짜 범위 조작이 무겁게 느껴지면 안 된다.

## 2. 시각 원칙

- 결과 리스트와 시간 슬롯의 밀도를 높인다
- 긴 설명 박스보다 상태, 시간, 가격, 원본 링크를 우선한다
- 공지와 푸터는 시각적 위계를 낮춘다
- 필터는 체크박스 나열보다 모바일 조작에 맞는 칩, 시트, 세그먼트 형태를 우선 검토한다
- MVP 필터는 상단 조건 요약 칩과 하단 필터 시트 조합으로 구성한다
- 결과는 큰 카드보다 날짜/시간 기준 촘촘한 리스트를 우선한다
- 비어 있음, 수집 실패, 오래된 데이터 상태를 명확히 구분한다

## 3. 인터랙션 원칙

- 조건 변경은 즉시 결과에 반영되어야 한다
- 브라우저 뒤로가기가 빈 화면으로 빠지면 안 된다
- 사용자는 원본 예약 페이지로 이동하기 전까지 현재 맥락을 잃지 않아야 한다
- 외부 예약 페이지 이동은 명확하지만 과하게 설명하지 않는다
- 새로고침, 수집 시각, 원본 링크는 신뢰도 판단에 필요한 만큼만 노출한다

## 4. 피해야 할 UI

- 진입 공지 모달
- 사이드바 메뉴 과다
- 첫 화면의 큰 안내 텍스트 박스
- 조건 검색마다 긴 로딩 스피너
- PC 관리자 도구처럼 보이는 대시보드
- 네이버 지도보다 못한 업체 소개 페이지
- 공연장 예약, 개인연습실, 커뮤니티가 섞인 포털형 메인 화면
- 큰 장소 카드가 화면 대부분을 차지하는 탐색형 레이아웃
