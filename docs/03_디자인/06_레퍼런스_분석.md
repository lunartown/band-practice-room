## 진행 경위

[05_레퍼런스_수집_기준](05_레퍼런스_수집_기준.md)까지 정리한 뒤, 별도의 경쟁사 앱 UI를 수집·분석하는 단계를 따로 수행하지 않았다. 대신 Claude Design 1차 시안(`assets/claude-design-2026-06-15/`)을 출발점으로 삼아 곧바로 [07_Figma_시안_브리프](07_Figma_시안_브리프.md) → [08_구조안_선택](08_구조안_선택.md)으로 넘어갔다.

이유:

- 제품이 "예약 가능 시간 검색 엔진"으로 좁게 정의되어 있어, 일반 합주실/예약 앱 UI를 폭넓게 분석하기보다 날짜 섹션 중심 결과 리스트라는 핵심 구조를 빠르게 시안으로 검증하는 편이 효율적이었다.
- 시안 기준 구조안을 바로 실제 웹(`src/web`)으로 구현하며 UI를 반복 개선하는 방식이 레퍼런스 정리보다 빨랐다.

## 후속

경쟁사 비교가 다시 필요해지면([04_경쟁사_분석](../01_기획/04_경쟁사_분석.md)과 연계) 이 문서를 채운다. 현재 MVP 범위에서는 별도 레퍼런스 분석을 두지 않는다.
