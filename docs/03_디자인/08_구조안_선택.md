# 구조안 선택

> 작성일: 2026-06-15 (갱신: 2026-06-22)
> 상태: 구현 완료 — A안 기준으로 `src/web` 구현·배포됨

## 1. 후보

| 안 | 설명 | 장점 | 단점 |
| --- | --- | --- | --- |
| A. 날짜 섹션 중심 | 날짜별 섹션 아래 예약 가능 슬롯을 시간순으로 나열 | 사용자가 합주 날짜를 먼저 정하는 실제 흐름과 잘 맞음. 30일 범위를 훑기 쉽고 공유 URL과도 잘 맞음 | 같은 시간대의 여러 선택지를 한 번에 비교하는 힘은 약함 |
| B. 시간대 중심 | 특정 날짜 안에서 시간대별로 가능한 합주실을 묶음 | "오늘 저녁 아무 데나"처럼 시간 우선 탐색에 강함 | 30일 전체 범위에서는 날짜 맥락이 약해지고 화면 구조가 복잡해짐 |
| C. 합주실별 묶음 | 합주실 단위로 가능한 시간을 묶어 표시 | 특정 합주실 선호가 강한 사용자에게 유용 | 검색 엔진보다 장소 포털처럼 보일 위험이 큼. MVP 핵심과 거리가 있음 |
| D. 다크 테마 | A안을 어두운 배경으로 변형 | 밤 시간대 사용이나 음악 서비스 분위기에는 맞을 수 있음 | MVP에서 우선순위가 낮고 기본 가독성 검증을 분산시킴 |

> 갱신 (2026-06-22): 첫 화면을 지역부터 고르는 **오픈 화면**으로 두기로 했다. 단, 이는 업체 소개·이미지가 있는 랜딩이 아니라 지역 선택 + 브랜드만 둔 경량 진입 상태이며, 지역을 고르면 즉시 아래 결과 구조로 전환한다. 재방문 시에는 마지막 조건을 복원해 오픈 화면을 건너뛴다. 따라서 아래 "검색 결과로 시작" 기준은 **결과 구조 자체**에 대한 기준으로 읽는다 (의사결정 로그 2026-06-22).

## 2. 선택 기준

- 첫 화면이 업체 소개가 아니라 검색 결과(또는 결과로 바로 이어지는 지역 선택)로 보여야 한다
- 사용자가 30일 범위의 빈 시간을 빠르게 훑을 수 있어야 한다
- 모바일에서 한 손으로 조건을 바꾸고 결과를 다시 확인하기 쉬워야 한다
- 결과 항목은 시간, 합주실명, 방 이름, 지역, 가격, 예약 이동을 한 줄 단위로 비교할 수 있어야 한다
- 장소 카드, 업체 소개, 큰 이미지, 긴 안내 문구는 배제한다
- 공연장 예약, 개인연습실, 커뮤니티, 공지 모달은 MVP 화면에 포함하지 않는다

## 3. 결정

MVP 기본 구조는 **A. 날짜 섹션 중심**으로 채택한다.

사용자가 실제로 합주 일정을 잡을 때는 "어느 합주실을 갈까"보다 "언제 비는 곳이 있나"를 먼저 확인하는 경우가 많다. 따라서 첫 화면은 날짜별 섹션을 기준으로 잡고, 각 날짜 안에서 시간순 슬롯을 촘촘하게 보여준다.

채택 구조:

- 상단: 서비스 타이틀
- 조건 영역: 현재 조건 요약 칩과 필터 버튼
- 요약 바: 빈 시간 개수와 정렬 기준
- 본문: 날짜 섹션 중심의 슬롯 리스트
- 슬롯: 시간, 이용 가능 길이, 합주실명, 방 이름, 지역, 가격, 예약 버튼
- 필터: 하단 bottom sheet
- 상태: 결과 있음, 결과 없음 (신선도/stale 상태는 표시하지 않음 — 의사결정 로그 2026-06-21)

보조 구조:

- **B. 시간대 중심**은 추후 "오늘/내일 빠른 탐색" 또는 특정 날짜 상세 보기에서 보조 뷰로 검토한다.
- **C. 합주실별 묶음**은 MVP 기본 화면에서 제외한다. 장소 선호 필터나 상세 화면을 만들 때만 다시 검토한다.
- **D. 다크 테마**는 MVP에서 제외한다. 기본 라이트 테마의 정보 밀도와 가독성을 먼저 검증한다.

## 4. Claude Design 시안 리뷰

원본 산출물:

- [RoomSearch.dc.html](assets/claude-design-2026-06-15/RoomSearch.dc.html)
- [SlotRow.dc.html](assets/claude-design-2026-06-15/SlotRow.dc.html)

채택할 점:

- 모바일 첫 화면을 검색 결과로 시작한 점
- 상단 조건 요약 칩과 하단 필터 시트 구조
- 날짜 섹션 중심 리스트를 기본안으로 둔 점
- 가격을 슬롯 안에 표시한 점
- 결과 없음 상태에서 조건 완화 액션을 제안한 점

수정할 점:

- 화면 제목 "빈 합주실"은 의미가 좁으므로 "빈 시간" 또는 "예약 가능 시간"으로 바꾼다
- 결과 개수는 "4곳"보다 "4개 시간" 또는 "4개 슬롯"에 가깝게 표기한다
- 시안의 신선도/stale 표시(배너·슬롯 상태·마지막 확인 시각)는 채택하지 않는다. 사용자에게 데이터 신선도를 노출하지 않는다 (의사결정 로그 2026-06-21)
- 필터 적용 버튼의 "결과 234건 보기"는 초기 MVP 규모에 비해 과장되어 보이므로 실제 슬롯 수 기준으로 표시한다
- 다크 테마는 보류한다
