# 시간 검색 UI 설계 결정

> 작성일: 2026-06-16 (갱신: 2026-06-22)

## 1. 문제 재정의

기존 UI(슬롯 78개 나열)의 근본 문제는 **DB view와 Intent view의 불일치**다.

- 기존 UI가 답하는 질문: "빈 슬롯이 몇 개 있나요?"
- 사용자의 실제 질문: "내일 저녁 7시 이후에 2~3시간 합주할 수 있는 방이 있나?"

합주는 퇴근 후 저녁이나 주말에 하기 때문에, 낮 시간대 슬롯은 사용자에게 노이즈다. 사용자는 원하는 시간대에 연속 블록이 있는지를 빠르게 파악하고 싶다.

## 2. 시나리오

### 1. 사용자 질문

#### 단순 질문

- 수요일이나 다음주 월요일, 저녁 7시 이후에 2시간 연속 가능한 합주실 방이 있나?
- 이번주 주말에 홍대에서 4시간 연속 합주할 수 있는 시간대가 있나?

#### 복합 질문

- 특정 조건으로 필터링 (장비, 인원수, 가격 등)
  - 단순 필터링만 하면 되므로 UI의 이슈는 아님

## 결론 (수정사항)

- 날짜 범위 UI는 필요 없다. 무조건 특정 날짜 골라서 하게 되어 있음. 달력 뷰를 띄워서 가능한 날짜 고르도록 해라
- 시간대도 필요 없다. 날짜 구간으로 조정하도록 함. 예를 들어 저녁 7시 ~ 10시
- 합주 시간 정할 수 있게 해라 1시간 2시간 3시간 등등
