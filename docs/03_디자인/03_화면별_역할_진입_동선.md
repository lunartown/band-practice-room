# 화면별 역할·진입·동선

> 작성일: 2026-06-15
> 상태: 초안

## 1. 화면 목록

| 화면 | 역할 | 진입 | 다음 행동 |
| --- | --- | --- | --- |
| 오픈 화면 | 브랜드 노출 + 지역 선택 진입점 | 첫 방문(저장된 조건 없음) | 지역 선택 → 결과로 전환 |
| 예약 가능 시간 검색 | 핵심 검색과 결과 확인 | 오픈 화면에서 지역 선택, 또는 재방문(저장된 조건 복원) | 필터 조정 또는 원본 예약 이동 |
| 필터 시트 | 조건 조정 | 필터 버튼, 날짜/시간 영역 탭 | 결과 즉시 갱신 |
| 원본 예약 이동 | 외부 예약 페이지로 이동 | 결과 항목의 예약 버튼 | 새 탭/외부 브라우저에서 예약 |
| 빈 결과 상태 | 조건에 맞는 결과 없음 안내 | 검색 결과 0건 | 조건 완화 |

## 1-1. 오픈 화면

### 역할

첫 방문자의 진입점이다. 브랜드(`합주실닷컴`)를 전면에 노출하면서, 합주실 탐색의 자연스러운 첫 질문인 "어느 동네에서 찾으세요?"로 지역을 고르게 한다.

### 화면 구성

```txt
브랜드
  - 합주실닷컴 워드마크 + 한 줄 태그라인

지역 선택
  - "어느 동네에서 찾으세요?"
  - 지역 칩 (탭 즉시 결과로 진입)
  - 전체 지역 둘러보기 (보조 텍스트 버튼)
```

### 동선·원칙

- 별도 라우트가 아니라 같은 화면의 한 상태다. 공유 URL·뒤로가기 모델을 깨지 않는다.
- 지역을 고르면 그 지역으로 즉시 결과 화면으로 전환한다.
- 마지막으로 사용한 조건을 로컬에 저장한다. 저장된 조건이 있으면 오픈 화면을 건너뛰고 바로 결과로 들어간다.
- 업체 소개·큰 이미지·긴 안내문은 두지 않는다. 브랜드와 지역 선택만 둔다.

## 2. 예약 가능 시간 검색 화면

### 역할

사용자가 합주실 빈 시간을 찾는 메인 화면이다. 첫 방문은 오픈 화면에서 지역을 고른 뒤 이 화면으로 들어오고, 재방문은 저장된 조건을 복원해 바로 이 화면으로 진입한다.

### 화면 구성

```txt
상단
  - 서비스명 또는 짧은 타이틀

조건 영역
  - 현재 조건 요약 칩
  - 필터 시트 진입 버튼

결과 영역
  - 날짜/시간 기준 그룹
  - 촘촘한 결과 리스트
  - 가격
  - 원본 예약 이동
```

### 결과 정렬

기본 정렬은 날짜/시간 기준이다.

```txt
날짜 오름차순
  → 시간 오름차순
    → 지역
      → 합주실
        → 방
```

### 진입 상태

초기 진입 시:

- 날짜 범위: 오늘부터 30일
- 시간대: 전체
- 지역: 전체
- 합주실: 전체
- 결과: 가까운 날짜부터 표시

## 3. 필터 시트

### 역할

검색 조건을 조정하는 화면이다. 모바일에서는 하단 시트가 적합하다.

MVP 필터 구조는 **상단 조건 요약 칩 + 하단 필터 시트**로 확정한다. 모든 필터를 상단에 펼치면 결과 공간이 줄어들기 때문이다.

### 필터

| 필터 | 형태 | 메모 |
| --- | --- | --- |
| 날짜 범위 | 캘린더 또는 빠른 선택 | 오늘, 이번 주, 이번 달, 직접 선택 |
| 시간대 | 칩/슬라이더 | 오후, 저녁, 심야 등 빠른 선택 |
| 지역 | 칩 | 홍대, 합정, 신촌 등 |
| 합주실 | 검색 가능한 목록 | 많아지면 검색 필요 |
| 가격 | 후순위 | MVP에서는 표시 우선, 필터는 후속 가능 |

### 원칙

- 조건 변경 후 별도 검색 버튼 없이 즉시 결과가 바뀌어야 한다
- 다만 모바일 성능상 필요하면 “적용” 버튼을 둘 수 있다
- 필터 상태는 화면 상단에서 압축 표시한다

## 4. 원본 예약 이동

### 역할

예약 확정은 외부 원본 페이지에서 수행한다.

### 동선

1. 사용자가 결과 항목을 확인한다.
2. 가격, 시간을 본다.
3. “예약 페이지”를 누른다.
4. 원본 예약 페이지가 열린다.

### 주의

- 외부 이동이라는 사실은 명확해야 한다
- 하지만 긴 설명문으로 공간을 차지하면 안 된다
- 버튼 문구는 짧게 둔다

후보:

- 예약 페이지
- 원본 예약
- 예약하러 가기

## 5. 빈 결과 상태

### 역할

조건에 맞는 예약 가능 슬롯이 없을 때 사용자가 다음 행동을 하게 한다.

### 표시

```txt
조건에 맞는 빈 시간이 없습니다

시간대를 넓히거나 지역을 추가해보세요
```

### 액션

- 시간 전체로 보기
- 지역 전체로 보기
- 날짜 범위 넓히기

> 수집 실패·신선도 상태는 사용자 화면에서 다루지 않는다. 수집에 실패한 합주실은 결과에서 조용히 제외하고, 신선도 관리는 내부 운영 영역으로 처리한다 (의사결정 로그 2026-06-21).

## 6. 뒤로가기

SPA 라우팅에서 브라우저 back이 빈 화면으로 빠지면 안 된다.

필터 상태는 URL query에 반영하는 것을 우선 검토한다.

예:

```txt
/?from=2026-06-15&to=2026-07-14&area=hongdae&time=evening
```

이렇게 하면 공유와 뒤로가기가 자연스러워진다.

## 7. 공유

MVP 공유 방식은 URL query 기반 조건 공유로 확정한다.

특정 슬롯 하나를 공유하는 기능은 후순위다. MVP에서는 사용자가 현재 조건 결과 화면을 공유할 수 있으면 충분하다.
