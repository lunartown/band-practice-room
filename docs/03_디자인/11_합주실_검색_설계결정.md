# 합주실 검색 설계 결정

> 작성일: 2026-06-27
> 상태: 구현 완료 — `src/web`에 반영. 대상: 클라이언트(서버 계약 기존 `studioId` 재사용)

## 1. 문제

합주실은 "맨날 가던 몇 곳"이라, 사용자는 흔히 **특정 합주실의 자리 여부**를 본다:
"그라운드 자리 있나? → 없네 → 하모닉스는?" 그런데 기존 UI엔 이를 푸는 길이 없었다.

- 합주실 이름 검색이 없었다(API `studioId`는 있으나 UI 미노출).
- 결과는 **자리 있는 합주실만** 나와서, 그라운드에 자리가 없으면 목록에서 사라진다 →
  "자리 없음"과 "원래 안 나옴"을 구분할 수 없다.

## 2. 결정 요약

- 상단바에 **돋보기 아이콘**을 두고, 탭하면 **검색 팝오버**(텍스트 입력 + 실시간 필터 목록)를 띄운다.
- 합주실은 **89곳(12개 지역)** 이라 지역 칩 같은 평면 목록은 무리 → 타이핑으로 좁히는 검색이 맞다.
- 합주실을 고르면 그 합주실만 조회한다(`studioId`). 비어 있으면 **그 합주실 이름으로 "빈 시간 없음"** 을 답해
  "없네"를 분명히 한다.
- 합주실 검색은 **세션에 영구 저장하지 않는다**. "그라운드만 보기"가 새로고침 뒤에도 남으면 사용자가 갇힌다.

## 3. UI

- **위치**: top-bar 우측(동기화 상태와 즐겨찾기 사이). 칩 행(시간대·날짜·지역·필터)을 건드리지 않아
  89곳 검색에 공간이 충분하고 결과-우선(NFR-003) 원칙도 지킨다.
- **팝오버**: 기존 `Popover` 재사용. 상단 텍스트 입력(autoFocus) + 아래 스크롤 목록.
  목록은 이름 가나다 정렬, 이름·지역명으로 필터. 맨 위 "전체 합주실"로 해제.
- **다중 선택**: 단골은 보통 여러 곳이라 체크 방식 다중 선택을 기본으로 한다. 팝오버는 고를 때 닫히지 않고
  배경 탭으로 닫는다. "전체 합주실"로 모두 해제.
- **활성 표시**: 고른 합주실마다 칩 행 아래에 `합주실명 ✕` 알약을 띄워(여러 개면 줄바꿈해 쌓임) 각각 ✕로 뺄 수 있다.
  돋보기 아이콘도 액센트로 켜진다.
- **빈 상태**: 좁힌 합주실에 결과가 없으면 일반 "조건 넓히기" 대신 합주실명(여러 곳이면 "선택한 합주실 N곳") +
  "빈 시간이 없어요" + "전체 합주실 보기"를 보여준다.

## 4. 동작 규칙

- **지역 무시**: 합주실을 고르면 지역 조건은 보내지 않는다. 지역과 합주실이 어긋나면 빈 결과가 나와
  "없네?"를 잘못 답하기 때문이다. (`getSlots`에서 `studioId`가 있으면 `areaIds`를 생략)
- **클라이언트/서버**: 검색 목록은 `getStudios()`로 전체 카탈로그를 한 번 받고, 좁힘은 `studioIds[]`
  쿼리 파라미터로 서버가 처리한다(`s.id = ANY(...)`). 기존 단일 `studioId`도 레거시로 유지한다.
- **다중 선택**: 단골 여러 곳을 한 화면에서 함께 본다. "그라운드 + 하모닉스"를 한 번에 조회해
  각자의 빈 시간을 비교할 수 있다.

## 5. 구현 위치

- `src/web/src/components/StudioSearch.tsx` (신규) — 검색 입력 + 필터 목록.
- `src/web/src/App.tsx` — 돋보기 버튼, `studioFilter` 상태, `getStudios` 로드, `getSlots`에 `studioId` 전달,
  활성 알약, 합주실 인지 빈 상태.
- `src/web/src/styles.css` — `.search-toggle`, `.active-filter-pill`, `.studio-search*`.

## 6. 향후

- 단골 합주실을 즐겨찾기/선호로 저장해 검색 없이 바로 다중 좁힘 적용.
- 정렬 옵션(문서 10)과 함께 쓰면 "전체 → 가나다순/가격순"과 "특정 합주실 좁힘"이 상호 보완한다.
