# 화면 범위

> 작성일: 2026-06-15
> 상태: 초안

## 1. v1 화면 원칙

MVP는 합주실 예약 가능 시간 검색 도구다. 따라서 화면도 여러 메뉴를 가진 포털이 아니라, 검색과 결과 확인에 집중한다.

첫 화면은 곧 검색 화면이어야 한다. 랜딩 페이지, 공지 모달, 업체 소개 페이지를 먼저 보여주지 않는다.

## 2. v1 포함 화면

| 화면 | 목적 | 포함 여부 | 메모 |
| --- | --- | --- | --- |
| 예약 가능 시간 검색 | 날짜/시간/생활권 조건으로 빈 방 찾기 | 포함 | 첫 화면 |
| 필터 시트 | 날짜 범위, 시간대, 생활권, 합주실 조건 조정 | 포함 | 모바일 하단 시트 또는 상단 필터 |
| 원본 예약 이동 | 선택한 방의 원본 예약 페이지로 이동 | 포함 | 별도 상세 페이지보다 이동 액션 중심 |
| 데이터 상태 표시 | 갱신 시각, stale, 수집 실패 표시 | 포함 | 신뢰도 보조 정보 |
| 빈 결과 상태 | 조건에 맞는 방이 없을 때 대안 제시 | 포함 | 조건 완화 유도 |
| 수집 실패 상태 | 특정 합주실 데이터 수집 실패 표시 | 포함 | 전체 실패처럼 보이면 안 됨 |

## 3. v1 제외 화면

| 화면 | 제외 이유 |
| --- | --- |
| 홈/랜딩 페이지 | 사용자는 설명보다 결과를 원함 |
| 합주실 상세 소개 페이지 | 네이버 지도/예약 대비 차별점 약함 |
| 공연장 예약 화면 | MVP 범위 아님 |
| 개인연습실 화면 | MVP 범위 아님 |
| 커뮤니티/게시판 | 핵심 검색 경험을 흐림 |
| 악보 검색 | MVP 범위 아님 |
| 밴드 활동 가이드 | MVP 범위 아님 |
| 공지사항 전체 화면 | 핵심 기능과 같은 depth에 두지 않음 |
| 로그인/마이페이지 | MVP에서 직접 예약/개인화 없음 |
| 관리자 화면 | 초기에는 내부 데이터 관리 방식으로 대체 |

## 4. 화면 구조 요약

```txt
예약 가능 시간 검색
  ├─ 상단 조건 요약 칩
  ├─ 하단 필터 시트
  │   ├─ 날짜 범위
  │   ├─ 시간대
  │   ├─ 생활권 필터
  │   └─ 합주실 필터
  ├─ 날짜/시간 기준 촘촘한 결과 목록
  ├─ 갱신 상태
  └─ 원본 예약 페이지 이동
```

## 5. 화면 밀도 기준

모바일 첫 화면에서 최소한 다음 정보가 보여야 한다.

- 현재 조회 날짜 범위
- 대표 시간대 필터
- 결과 일부
- 각 결과의 날짜, 시간, 합주실, 방, 가격
- 마지막 확인 시각 또는 상태

첫 화면에서 피해야 할 것:

- 서비스 설명 박스
- 큰 공지 모달
- 사이드바 메뉴
- 업체 소개 카드
- 긴 도움말 문구
- 큰 장소 카드 중심 레이아웃

## 6. 후속 검토 화면

MVP 이후 검토:

- 합주실별 공개 페이지
- 자주 보는 조건 저장
- 즐겨찾는 합주실
- 가격 비교 뷰
- 제보/수정 요청 화면
- 운영자 데이터 보정 화면
