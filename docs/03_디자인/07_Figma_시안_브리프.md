# Figma 시안 브리프

> 작성일: 2026-06-15
> 상태: 초안

## 1. 목적

합주실 예약 가능 시간 검색 서비스의 MVP 모바일 화면 시안을 만든다.

목표는 예쁜 포털이 아니라, 사용자가 30일 범위의 예약 가능 시간을 빠르게 훑고 조건을 즉시 바꿀 수 있는 검색 도구를 설계하는 것이다.

## 2. 제품 방향

- MVP는 합주실만 다룬다
- 공연장 예약, 개인연습실, 커뮤니티, 악보, 밴드 활동 가이드는 없다
- 첫 화면은 검색 결과 화면이다
- 날짜/시간 기준 촘촘한 리스트가 중심이다
- 필터는 상단 조건 요약 칩 + 하단 필터 시트로 구성한다
- 예약 확정은 원본 예약 페이지에서 한다
- 가격과 마지막 확인 시각을 결과에 표시한다

## 3. 포함할 화면

### 화면 1. 예약 가능 시간 검색

모바일 기본 화면.

포함:

- 상단 타이틀
- 전체 갱신 상태
- 조건 요약 칩
- 날짜/시간 기준 결과 리스트
- 결과 항목: 날짜, 시간, 합주실, 방, 가격, 마지막 확인 시각, 예약 페이지 버튼
- 빈 결과 상태
- 일부 수집 실패 상태

### 화면 2. 필터 시트

하단 시트.

포함:

- 날짜 범위
- 시간대
- 생활권
- 합주실
- 초기화
- 닫기 또는 적용

### 화면 3. 빈 결과 상태

조건에 맞는 예약 가능 시간이 없을 때.

포함:

- 짧은 상태 문구
- 조건 완화 액션
- 날짜 범위 넓히기 또는 시간 전체 보기

### 화면 4. 오래된 데이터/stale 상태

마지막 수집 시간이 오래된 결과.

포함:

- 마지막 확인 시각
- 원본에서 다시 확인 필요하다는 낮은 위계의 표시
- 원본 예약 페이지 버튼

## 4. 화면에서 강조할 정보

우선순위:

1. 날짜
2. 시간
3. 합주실 이름
4. 방 이름
5. 가격
6. 원본 예약 페이지 이동
7. 마지막 확인 시각

후순위:

- 업체 소개
- 상세 주소
- 장비 설명
- 긴 안내 문구
- 공지

## 5. 톤

- 빠른 검색 도구
- 모바일 친화적
- 과하게 감성적이지 않음
- PC 관리자 대시보드처럼 보이지 않음
- 카드형 포털보다 촘촘한 리스트 중심
- 설명보다 결과 중심

## 6. 제약

- 진입 모달 금지
- 랜딩 페이지 금지
- 사이드바 메뉴 금지
- 공연장/개인연습실/커뮤니티 메뉴 금지
- 긴 설명 박스 금지
- 큰 장소 이미지 카드 중심 레이아웃 금지
- 조건 검색마다 긴 로딩을 전제로 한 UI 금지

## 7. 시안 요청 문장

```txt
모바일 우선 합주실 예약 가능 시간 검색 서비스의 MVP 화면을 디자인한다.

첫 화면은 랜딩 페이지가 아니라 검색 결과 화면이다. 사용자는 최소 30일 범위의 합주실 예약 가능 시간을 날짜/시간 기준으로 빠르게 훑는다.

상단에는 현재 조건을 요약하는 칩을 두고, 상세 필터는 하단 시트에서 조정한다. 결과는 큰 장소 카드가 아니라 촘촘한 리스트로 구성한다. 각 결과에는 날짜, 시간, 합주실 이름, 방 이름, 가격, 마지막 확인 시각, 원본 예약 페이지 이동 버튼이 있어야 한다.

공지 모달, 사이드바, 긴 설명 텍스트, 공연장/개인연습실/커뮤니티 메뉴는 넣지 않는다.
```
